# 🤝 Contributing Guide

Merci de contribuer à ce projet !
Pour garder un code propre et éviter le chaos en équipe, merci de suivre ces règles.

---

## 🚫 Règle principale

❌ **NE JAMAIS travailler directement sur la branche `main`**

---

## 🌿 Workflow à suivre

### 1. Cloner le projet

```bash
git clone <repo-url>
cd <project>
```

---

### 2. Créer une branche

Toujours créer une branche pour chaque nouvelle fonctionnalité ou correction.

```bash
git checkout -b feature/nom-de-la-fonctionnalite
```

Exemples :

* `feature/ajout-enseignant`
* `feature/login`
* `fix/bug-formulaire`

---

### 3. Faire les modifications

Travaillez uniquement dans votre branche.

---

### 4. Ajouter et commit

```bash
git add .
git commit -m "feat: ajout fonctionnalité enseignant"
```

---

### 5. Push vers GitHub

```bash
git push origin feature/nom-de-la-fonctionnalite
```

---

### 6. Créer une Pull Request

* Aller sur GitHub
* Cliquer sur **"Compare & pull request"**
* Décrire ce que vous avez fait
* Demander une review

---

## 🔍 Bonnes pratiques

✅ Écrire des messages de commit clairs
✅ Travailler sur une seule fonctionnalité par branche
✅ Tester son code avant de push
✅ Lire les changements des autres avant de fusionner

---

## ⚠️ Interdictions

❌ Push direct sur `main`
❌ Travailler sans branche
❌ Casser le projet sans tester
❌ Faire des commits vides ou incompréhensibles

---

## 💬 Communication

Si vous êtes bloqué :

* Demandez de l’aide à l’équipe
* Ne restez pas bloqué seul

---

Merci de respecter ces règles 🙌
Un bon workflow = un projet réussi 🚀
